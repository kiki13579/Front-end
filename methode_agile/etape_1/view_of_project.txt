A - Contexte du projet :
    Vous intégrez une start-up spécialisée dans la diffusion de messages spirituels et horoscopiques.
    Votre mission est de concevoir et de développer une application web frontend complète pour présenter un horoscope quotidien.
    Le projet, intitulé "Horoscope Oracle du Jour", part de zéro, sans aucun fichier de base fourni.
____________________________________________________________________________________________________________________________________
B - Objectifs du projet :
    Objectif Principal : Créer une application web frontend complète et fonctionnelle qui se connecte à une API externe 
        pour afficher les 12 messages de l'horoscope du jour.

    Objectifs Secondaires :
        Offrir une expérience utilisateur qui soit à la fois fluide, moderne et responsive.
        Mettre en place une interface claire, élégante et visuellement attractive.
        Assurer que le code soit bien structuré, commenté et suive les bonnes pratiques professionnelles.
        Déployer l'application en ligne pour la rendre accessible

D - Périmètre et Contraintes du projet
    Fonctionnalités principales :
        Un bouton unique pour déclencher l'affichage des 12 messages.
        Affichage dynamique des horoscopes sous forme de cartes visuelles générées en JavaScript.
        Présence d'un indicateur de chargement (loader) pendant l'appel à l'API.
        Gestion et affichage d'un message d'erreur clair en cas d'échec de la requête.
        Animations CSS légères au survol des cartes.

    Contraintes techniques :
        Technologies imposées : HTML5 sémantique, CSS3 (Flexbox, Grid, Media Queries) et JavaScript natif (vanilla JS).
        Appel API : Utilisation obligatoire de fetch() pour effectuer une requête POST avec un Content-Type: application/json et une clé d'API secrète.
        Responsive Design : L'application doit être parfaitement fonctionnelle sur mobile, tablette et ordinateur.
        Structure de projet : Une arborescence de fichiers claire et professionnelle est requise (/assets/css, /assets/js, etc.).
